Self-Hosting: eigene Client ID

Nur für Self-Hosting nötig

Auf shift.fr4iser.com ist eine Client ID bereits hinterlegt — dort reicht „Mit Google verbinden“. Keine Einrichtung nötig.

Hostest du das Tool selbst (eigene Domain oder localhost), funktioniert die mitgelieferte ID meist nicht: Google erlaubt OAuth nur für freigeschaltete Ursprünge. Dann brauchst du eine eigene Client ID.

Wo eintragen?

In src/config.json das Feld googleClientId setzen (kein Secret — darf im Repo stehen). Vorlage: src/config.example.json.

Kurz-Anleitung (Google Cloud)

  1. Google Cloud Console → Projekt anlegen
  2. API aktivieren: Google Calendar API
  3. OAuth-Zustimmungsbildschirm (z. B. Extern) — App-Name + Support-Mail
  4. Anmeldedaten → OAuth-Client-ID → Typ Webanwendung
  5. Autorisierte JavaScript-Ursprünge = exakt deine URL, z. B. http://localhost:8080 oder https://deine-domain.de
  6. Client ID in src/config.json speichern und Seite neu laden

Hinweise

  • Kein Client-Secret im Frontend — wird nicht gebraucht.
  • Sync läuft Browser ↔ Google; der Hosting-Server sieht keine Kalenderdaten.
  • Ohne Google geht’s auch: PDF konvertieren und .ics / CSV exportieren.
